# DESIGN — Kids & Fun מערכת ניהול לקוחות + בוט שירות

> קובץ זה מסכם את האפיון המלא של המערכת.
> **יש לקרוא קובץ זה לפני כל שינוי בקוד.**

---

## מה המערכת עושה

מערכת CRM + בוט WhatsApp לניהול צהרון וקייטנה של Kids & Fun.
מחליפה תהליכים ידניים (2-3 ימי עבודה/חודש) במערכת אוטומטית.

---

## ארכיטקטורה

```
הורה / עובד
    ↓ WhatsApp
בוט מסוואג (intent detection + flows)
    ↓ API סנכרון
דשבורד CRM (Next.js + Supabase)
    ↓ אינטגרציות
PayPlus (תשלומים) + חשבונית ירוקה (קישורי תשלום) + EasyDoc (טפסים)
```

## 7 מסלולי הבוט (מהאפיון)

### מסלול 1: רישום לצהרון
**כניסה:** הורה פונה לרישום
**תהליך:**
1. מילוי טופס רישום דינמי (מותאם לסוג המוצר)
2. בדיקת מקום פנוי
3. בדיקת סטטוס תשלום → הפניה אם נדרש
4. אישור ושיוך למסגרת
**סטטוסים:** רישום מלא / רישום חלקי / ממתין לאישור
**קיים:** ✅ חלקית — טופס `/register`, שמירה ב-Supabase

### מסלול 2: ביטול רישום
**כניסה:** הורה / עובד מבקש ביטול
**תהליך:**
1. קבלת בקשת ביטול
2. בדיקת זכאות לביטול (תקופה + הנגאים)
3. חישוב זיכוי/החזר לפי מדיניות
4. אישור הביטול
5. עדכון מערכת ותיעוד
**כלל מרכזי:** ביטול עד ה-15 לחודש = זיכוי לחודש הנוכחי. אחרי ה-15 = זיכוי לחצי חודש הבא.
**סטטוסים:** ביטול מאושר / ממתין לאישור / נדחה
**קיים:** ❌ חסר — רק routing בסיסי, אין חישוב זיכוי אוטומטי

### מסלול 3: רישום לקייטנה / מאוחר
**א׳ — לפני סגירת המועד (בוט מסייע):**
- תרחיש 1: רישום מהיר → קישור + תשלום אוטומטי
- תרחיש 2: בדיקת רישום → הורה מאשר
- תרחיש 3: רישום בליווי → צוות מלווה + תשלום
**ב׳ — אחרי סגירת המועד (ידני 100%):**
- בוט אוסף שם ילד + כיתה + מסגרת → מתעד ב-CRM
**קיים:** ❌ חסר

### מסלול 4: שאלות לו"ז / חגים
**תהליך:** קבלת פנייה → ניתוח שאלה → בדיקה ב-DB → יצירת תשובה אוטומטית → מעקב ותזכורות
**קיים:** ❌ חסר — אין DB של חגים/שעות

### מסלול 5: איסוף מוקדם
**תהליך:** קבלת בקשה → אימות הרשאה וזהות → בדיקת לוח זמנים → עדכון צוות גננות
**קיים:** ❌ חסר

### מסלול 7: מעקב סטטוס תשלום
**תהליך:** קבלת פנייה → בדיקת סטטוס במערכת → הצגת סטטוס → הפניה אם נדרש → עדכון וסיום
**קיים:** ✅ קיים

### מסלול 7א: כשל תשלום — פנייה מצד ההורה
**תרחיש הכי קריטי!**
הורה יוזם פנייה כי הרגיש שהתשלום נכשל.
בוט שואל: "הכל בסדר? הבנק ניסה לחייב את הכרטיס אבל לא הצלחנו. האם הכרטיס שלך החלפת?"
4 תרחישים: פרטים חדשים / תזכורת מאוחרת / שינוי אמצעי / בעיה בתאריך
**קיים:** ✅ קיים חלקית

### מסלול 7ב: כשל תשלום — פנייה יזומה מהמערכת
בוט שולח הודעת פתיחה ללקוח כאשר מזהה כשל.
**קיים:** ✅ קיים
